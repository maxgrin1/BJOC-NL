Keeping Score with Global Variables

Op deze pagina, ga je een globale variabele gebruiken om je score in het Alonzo spel bij te houden.

In je Raad-het-getal spel, heb je een script variabele gebruikt om informatie in op te slaan (de waarde van het geheime getal) die maar bij één script van toepassing was. Om de score bij te houden in het Alonzo spel, meer dan één scripts zullen dan nodig zijn om toegang te krijgen tot bepaalde informatie, dus gebruik je een globale variabele.

Een globale variabele is een variabele die bruikbaar is door alle scripts in het programma.

Wanneer gebruik je globale variabelen?

In de meeste gevallen, zijn script variabelen een betere keuze omdat ze niet veranderd kunnen worden door een andere gedeelte van het programma. Op deze manier verkleinen ze de kans op bugs. Maar als meerdere scripts toegang nodig hebben tot een variabele of als informatie uit een variabele opgeslagen moet worden met je document, gebruik dan een globale variabele.

  1. Open je H1L1-Alonzo project, speel het spel en bekijk de code net zo lang totdat je weer zeker weet hoe alles werkt.
  2. "U2L1-ClickAlonzo"
    3. Zorg dat je je project opnieuw opslaat onder een andere naam met "H2" in het documentnaam. Als je dit werk opslaat onder je oude naam, zal Snap! je project overschrijven. Gebruik "Opslaan als..." om het nieuwe bestand op te slaan onder een nieuwe naam. Als je een ouder project laadt en op het punt staat een grote verandering door te voeren, probeer er dan een gewoonte van te maken het eerst op te slaan, voordat je iets verandert.
  3. Create a global variable named score. Here's how.

    4. Maak een Globale Variabele

    1. Klik in het Variabelen palette. (Het is geen blok; je kan dit niet slepen in je script gedeelte.)
    2. Type de titel voor je variabele. In dit geval, is het score. Klik OK.
    Je kunt of gebruiken. In pseudocode ziet deze code er zo uit: 
    score ← score + 1
  4. Gebruik de score variabele om de score van de speler bij te houden:
    1. Zet score op 0 in het begin van het spel.
    2. Zorg dat het programma de score verandert met 1, wanneer de sprite wordt aangeklikt.
  5. Test en debug. Speel het spel net zolang totdat je zeker weet dat je score variabele werkt.
  1. Op dit moment, stopt het spel nooit, maar dat ga je veranderen! Kies een score getal die je zult gebruiken om te testen of de speler gewonnen heeft.
  2. Gebruik een conditionele om te besluiten of de speler die score heeft bereikt. Als dit het geval is:
    1. Zorg dat Alonzo stopt met rondspringen.
    2. Zorg dat Alonzo weer tevoorschijn komt. (Je moet dan het ghost effect resetten.)
    3. Feliciteer de speler voor een paar seconden.
  3. Test en debug. Speel het spel om de beurt, en fix bugs.

    Debugging Tip: Weergeven van variabelen

    Globale variabelen hebben check-boxes die de weergave van de variabele kijker hebben (zie hieronder). Dit maakt het makkelijker om errors in je programma te spotten.

 
  1. Wanneer de sprite beweegt zonder aangeklikt te worden, zorg dat je score met 1 omlaag gaat.
