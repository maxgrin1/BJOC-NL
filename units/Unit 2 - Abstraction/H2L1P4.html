Importeer Groet Spelers in een ander programma

Wat je zult tegen zult komen, is dat je al eens een stukje code hebt geschreven die in een later script goed van pas zou komen.

Op deze pagina, ga je je code groet speler blok kopiëren in je Raad-het-getal spel, zodat je het daar kunt gebruiken.

    Dit project is van Hoofdstuk 1 Les 2 Pagina 4: Leer de namen van spelers.
  1. Open je H1L2- Groet Speler project, run het een paar keer en bekijk je code net zo lang totdat je weer zeker bent hoe het werkt. (Op het moment dat je wat namen in je spelers lijst hebt, test dan met wat namen die op de lijst staan en die niet op de lijst staan.)
  2. Fix alle problemen met groet speler. Op het moment dat het correct werkt, exporteer het. (Instructies hieronder.)

    Exporteer blokken

    1. Kies "Exporteer blokken..." van het Snap! menu ().
    2. Selecteer enkel de blokken die je wilt exporteren (in dit geval, enkel groet speler), en klik "OK." Een XML document gaat nu downloaden.
      3. De Snap! Reference Manual vertelt meer over exporteren en importeren.
  3. En dan nu, open je je H2L1-RaadHetGetal project, en importeer je groet speler blok.

    Importeer blokken

    1. Kies "Importeer..." van het Snap! menu ().
    2. Vind de XML file die je eerder geëxporteerd hebt (waarschijnlijk op je desktop in de 'download' map) en klik Openen.
    3. Vind de blokken die je wilt. Deze zullen aan het eind van de palette die deze bevat verschijnen.
    4. Klik het geïmporteerde blok om ervoor te zorgen dat alles het goed doet. (In dit voorbeeld, zul je waarschijnlijk een error tegenkomen nadat je een naam hebt ingevoerd.)
  4. Look at the code for greet player. Can you determine what's wrong?

Debuggen van afhankelijken

Soms, als je een script neemt uit het programma waarin je dit hebt gebouwd, en het dan ergens anders uitprobeert, gaat er iets kapot. Het kan zijn dat het script afhankelijk is van een speciaal blok in het nieuwe programma, of (zoals in dit geval) dat het script afhankelijk is van een globale variabele dat niet gedefinieerd is in het nieuwe programma.

Om dit probleem op te lossen zodat je het geïmporteerde blok goed kunt gebruiken, zul je je globale variabele moeten definiëren in je nieuwe programma. Afhankelijken zijn een reden om eerder een script variabele te gebruiken dan een globale variabele wanneer mogelijk. Dit omdat ze gedefinieerd zijn in hun scripts, en niet kwijtraakt wanneer je het script naar een andere plek verplaatst.

  1. Maak de ontbrekende globale variabele, spelers lijst, en zorg dat het programma het initieert als lege lijst.
    Om een lege lijst aan te maken, klik op de linker pijl op het lijst-blok, zodat er geen nieuwe invoer plekken zijn.

    2. Als spelers lijst is geïnitieerd elke keer wanneer het spel opnieuw start, zal het spel nooit een naam onthouden. Dus, je zult een ander moment nodig hebben om het spel te starten dan het initiëren van de spelers lijst. Bijvoorbeeld, je kunt de groene vlag gebruiken om de lijst te initiëren, en de spatiebalk gebruiken om het Raad-het-getal spel te starten.

  2. Test en Debug. Je programma dient
    • de spelers te groeten en hun namen te onthouden;
    • het Raad-het-getal spelt telkens te starten met een nieuw geheim getal.
    Fix alle problemen met je code.
    Is er een probleem met de variabele?
    • Check de variabele initialisatie (daar waar het van origine is gebouwd): Staat de code waar het hoort te staan? Doet de code datgene waarvoor het is bedoeld?
    • Check waar de variabele is gebruikt: Gebruik je de variabele die je bedoelt? Gebruik je het op de manier zoals je verwachtte?
